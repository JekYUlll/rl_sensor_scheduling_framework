# 当前强化学习调度路线说明（口语版）

这份说明按一条连续主线来讲：从数据怎么来，到策略怎么学，再到结果怎么判断。

## 1. 先看这套系统现在已经做成了什么

仓库现在已经完整实现了五层能力。

第一层是环境层。系统会先生成一条共享的高频真值时间序列，里面有风、温湿压、辐射、雪面温度、雪粒信息和风吹雪通量。所有调度策略都在同一条真值上运行，所以对比是公平的。

第二层是观测层。每个传感器有自己的测量变量、噪声、刷新节奏、稳态功耗和启动峰值功耗。策略给出开关决定以后，系统会按传感器特性生成“当下真能拿到的观测”，并记录功率轨迹。

第三层是估计层。系统用在线状态估计器把“部分观测”转成“连续状态认知”，同时维护不确定性、各传感器新鲜度、覆盖率、预算使用比例、上一时刻开关等信息。强化学习看到的状态就是这一层的摘要。

第四层是调度学习层。系统已经同时实现了规则策略、值函数策略、带约束的值函数策略、策略梯度策略，并且可以在统一环境里训练、验证、测试。

第五层是预测评估层。每个调度策略都会回放生成一份自己的训练数据，再交给同一组时序预测模型训练。最后统一汇总误差、形状一致性、相关性、节电和约束违约率。

## 2. 当前默认开启了什么，没开启什么

当前默认开启的内容：

- 主任务直接误差驱动的调度奖励；
- 启停代价与覆盖率惩罚；
- 动作可行域里的硬约束控制：瞬时功率上限、启动峰值上限、安全裕度；
- 约束版策略中的长期功率与能量约束处理；
- 全链路训练、回放、下游预测、聚合评估。

当前默认未开启的内容：

- 冻结预测器提供的每步前瞻奖励；
- 预测奖励在训练损失中的权重（默认是零）；
- 端到端的“调度器和下游预测器联合训练”。

还有一个关键现状：系统会把奖励预测器的预训练步骤跑一遍，这一步当前处于“准备状态”。默认主训练不会把它接进回报计算。

## 3. 现在真正执行的主路线

主路线是“先学调度，再看预测”。流程按时间顺序是这样走的。

先在训练段反复做交互。每一步，策略拿到当前状态摘要，给出传感器偏好。约束投影器把这个偏好变成一个满足功率与峰值限制的可执行开关子集。环境推进一步，返回这一刻可观测信息。估计器完成状态更新，系统计算当步损失并取负值作为回报，策略继续学习。

这里的当步损失核心由三块组成：主任务状态误差、开关切换代价、覆盖率惩罚。功率项主要通过约束层控制，约束版策略还会在回合尺度上更新约束代价的拉格朗日权重。

训练过程中会周期性做验证，用验证段指标挑选最优模型。训练结束后，用测试段做独立评估，确认奖励、功率、峰值违约、覆盖率等调度层指标。

接下来进入预测层。系统把每个调度策略在全时段回放一遍，导出该策略对应的数据集。然后用同一组预测模型分别训练，得到各策略在相同模型条件下的预测表现。

最后进入汇总层。系统统一输出误差指标、形状指标、相关性指标、节电指标、约束违约指标，并与全开上限做相对比较，形成横向结论。

## 4. 为什么这条路线可以先训练调度，再训练下游预测

因为当前调度训练信号已经完整可用。系统在每一步都能直接算出主任务状态误差和工程代价，所以强化学习可以独立收敛。

下游预测在这条路线里承担“质量检验器”的角色。它检验的是：这套调度生成的数据，是否还保留了对未来预测有价值的信息。这样做的好处是训练稳定、职责清晰、对比公平。

## 5. 现在该怎样读结果图和结果表

第一看误差。误差决定可用性底线。

第二看节电。节电决定部署价值。

第三看违约率。违约率决定工程可执行性，尤其要盯峰值相关约束。

第四看相对上限差距。全开策略给出理论参考点，其他策略的目标是用更低能耗逼近这个参考点。

第五看曲线形状和相关性。很多时候数值误差接近，时序形状差异会很大，控制场景里这部分差异同样重要。

把这五类信息合在一起，你就能判断一条策略到底是“省电但失真”，还是“省电且保持可用预测能力”。

## 6. 现在这套仓库的定位

现在的仓库定位很清晰：

- 它已经是完整可运行的“功率受限多传感器调度 + 状态估计 + 下游预测评估”平台；
- 它的默认主路线强调训练稳定与工程约束优先；
- 它已经保留了前瞻奖励增强入口，后续可以在可控实验里逐步启用并验证收益。

这就是当前版本真实在跑的路线。
