# 结论（目标版草稿）

> 本文档是按“优化后的合理目标状态”撰写的论文结论草稿，用于明确论文最终希望支撑的结论口径。它不是对当前单轮实验结果的直接照抄，而是建立在后续输入表达、特征工程、状态表示与多随机种子验证进一步完善之后的目标性表述。尤其是，`full_open` 在理论上应被视为信息最充分的 oracle 上限基线，因此最终论文结论应以“预算受限策略能否在明显节电的同时逼近 `full_open`”为核心，而不应把个别模型上偶然超过 `full_open` 的现象写成主结论。

## 结论

本文面向功耗受限场景下的多传感器调度问题，构建了一套统一的实验框架：首先在共享高频真值环境上生成包含基础气象、辐射、雪面温度、风吹雪粒径/粒子平均速度以及风吹雪通量的多变量序列；随后在统一功率预算下训练或评估不同调度策略；再将各调度策略回放到同一真值环境上，生成对应的估计状态数据集；最后在多个常见时序预测模型上分别训练并比较下游预测精度。该框架使得“调度是否真正保留了对目标预测有用的信息”可以在统一实验条件下得到检验。

在论文所追求的合理最终结论口径中，`full_open` 应始终被解释为最充分观测条件下的 oracle 上限，即它代表“不做调度、信息最完整”时的预测精度参考。相应地，各类预算受限调度策略的价值，不应被表述为是否“击败” `full_open`，而应被表述为：在满足总功率约束的前提下，能够以多小的精度代价逼近 `full_open`。因此，最终论文的核心结果应围绕功耗—精度折中展开：如果某一调度策略能够在节电约 40%–60% 的条件下，将相对 `full_open` 的 RMSE 增幅稳定控制在较低范围内，那么该策略就证明了其工程价值。

基于这一口径，本文应得出的主结论是：调度设计必须围绕最终预测任务展开，而不能仅仅优化状态估计层面的 surrogate 指标。早期实验已经表明，单纯依赖 `trace(P)` 或其它估计导向代价，会导致学习型调度器优先保留对协方差最有利、却未必对目标预测最关键的观测，从而出现“估计更优但预测更差”的目标错配。经过奖励重构之后，调度器的优化目标应更多体现风吹雪通量预测本身，而功耗应主要通过硬约束控制、而不是作为压倒性的 reward 项主导训练。这样才能使调度器真正学到“在有限功率下保留最有预测价值的信息”。

在这一框架下，规则型调度方法仍应被视为强基线，而不是陪衬。特别是 `round_robin` 与 `info_priority` 两类方法，本质上分别对应“覆盖公平性优先”和“目标相关性优先”两种具有明确工程含义的设计思想。最终论文若要证明学习型调度器的必要性，应当以这些强规则基线为比较对象，而不是仅与 `random` 或 `periodic` 这类较弱策略对比。也就是说，学习型方法的贡献不应被写成“任何时候都优于启发式策略”，而应被写成：在相同功率约束下，能够更稳定地接近 `full_open` 上限，并在多个预测模型上保持一致的精度优势或更优的平均折中表现。

针对学习型调度器，合理的论文结论应是：当 reward 以预测目标为中心重新设计，并且状态表示中充分编码观测新鲜度、驱动变量与响应变量之间的结构关系后，基于 DQN 的调度器应当能够显著缩小与 `full_open` 的差距，同时优于主要规则基线中的至少一部分。换言之，强化学习在这里的价值不在于理论上“创造比全开更多的信息”，而在于在固定功率预算下更有效地分配观测机会，优先保留对目标预测最有贡献的传感器组合与采样时机。如果最终结果表明学习型调度器在大多数模型上都能把相对 `full_open` 的误差增幅压缩到较小范围，同时实现显著节电，那么这将构成本文最有力的结论之一。

对于下游预测模型，本文的最终合理结论不应是“某个模型偶然比 `full_open` 更准”，而应是：当输入表达被充分设计后，更复杂或更具归纳偏置的预测器能够更有效地利用全开观测带来的额外信息，从而使 `full_open` 在理论上应有的上限优势真正体现出来。若部分简单模型在个别实验中出现轮询优于全开的现象，这更应被解释为当前输入表示尚未充分利用全部观测，而不是调度本身创造了更高的信息上限。因此，论文应把这种现象写成输入表示与特征工程的改进动机，而非最终结论。

引入物理约束预测器（例如轻量 PINN）时，合理的结论口径应当是：物理约束能够提升模型对已有信息的利用效率、增强对异常预测的约束能力，但它不能替代调度端对关键信息的保留。也就是说，PINN 的角色应被定位为“提高下游利用效率”的补强手段，而不是对调度缺陷的补救。最终论文若要呈现这一部分，应强调：物理先验与调度策略是互补关系，前者提升预测器的利用能力，后者决定可利用信息的上限。

综合来看，本文最终希望支撑的三点结论应当是：第一，`full_open` 是信息最充分的 oracle 上限，预算受限调度策略的评价标准应是“以多小代价逼近该上限”，而不是在个别模型上偶然超过它；第二，规则型调度是必须严肃对待的强基线，学习型调度器只有在预测导向 reward 和更合理输入表达下，才有可能稳定体现优势；第三，传感器调度问题的关键不只是“获取更多数据”，而是“如何把有限观测预算转换成对目标预测最有效的信息表示”。这一定义使得多传感器调度、状态估计与下游时序预测之间形成了一个闭环清晰、可解释且工程上有价值的研究问题。

## 局限性与后续工作

即便在优化后的目标口径下，本文仍应诚实陈述其边界。首先，当前状态估计器本质上仍是数据驱动的线性高斯 surrogate，而非严格的物理风吹雪动力学模型；因此，它更适合作为调度实验中的统一 belief-state 生成器，而不应被过度解释为真实物理机制。其次，本文的主要目标变量仍聚焦于 `snow_mass_flux_kg_m2_s`，这使得某些 target-aware 启发式策略天然更强；后续若要扩展任务范围，可以考虑在保持主目标不变的前提下，引入少量关键驱动变量作为辅助目标，而不是简单地对所有气象变量平均建模。再次，要想严格支撑“学习型策略优于规则基线”的论断，仍需要补充多随机种子、多 horizon 以及统计显著性检验。

因此，后续工作应沿以下方向推进：其一，继续改进输入表示与特征工程，使 `full_open` 的信息优势能够被下游模型充分利用，例如加入观测新鲜度、观测掩码、驱动—响应交互项与物理启发派生特征；其二，进一步对 reward 进行任务对齐，使学习型调度器真正围绕 forecasting gain 而非协方差缩减进行优化；其三，在现有框架上开展更系统的多种子与多模型实验，最终把本文的结论从“现象级合理”推进到“统计上稳健可信”。
