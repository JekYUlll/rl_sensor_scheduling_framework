# RL Sensor Scheduling Framework

A standalone framework for power-constrained multi-sensor scheduling with RL.

## Scope
- Environment evolves over time (default linear Gaussian).
- Scheduler selects sensor subsets under per-step and long-term power constraints.
- Estimator (default Kalman Filter) updates state belief from selected observations.
- RL policy (DQN in Phase 1) optimizes long-term sensing quality vs power.
- Forecasting models are evaluated on trajectories generated by scheduling policies.

## Quick start
```bash
cd rl_sensor_scheduling_framework
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/01_train_rl_scheduler.py \
  --env_cfg configs/env/linear_gaussian_case.yaml \
  --sensor_cfg configs/sensors/linear_gaussian_sensors.yaml \
  --estimator_cfg configs/estimator/kalman.yaml \
  --scheduler_cfg configs/scheduler/dqn.yaml \
  --run_id phase1_debug
```

## Project phases
- Phase 1: linear Gaussian env + KF + baselines + DQN.
- Phase 2: business adapter interfaces.
- Phase 3: windblown case integration.
- Phase 4: forecasting evaluation under learned policy.
- Phase 5: posthoc and statistical analysis.
